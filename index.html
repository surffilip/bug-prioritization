How to prioritize that bug

So many bugs so little time, how to know what bug is important, and how to prioritize among them. This is a little tool that could help in that decision making

the Bug prioritization Matrix
Who's affected / What are the consequenses 🤷‍ none 🎨 Minor aesthetic error (alignment, color etc) ❕ Major aesthetic error, strange interaction, bad performance 💣 Broken things, important content isn't shown, important buttons don't work. Awful performance 💥 Extremely critical. Site is down, nothing works
🤷‍ No one 🗑🗑🗑🗑🗑
One person or only admins/devs 🗑🗑🧊🔥⚠
A few users or only editors 🗑🧊📜🔥⚠
Many users/visitors 🗑📜🏃‍⚠⚠
All users 🗑📜🔥⚠⚠

Legend

  • Wont fix
  • Put in icebox
  • Put in backlog (prioritize in next sprint start)
  • Put in current sprint, release in planned sprint release
  • Put in current sprint, Hotfix when done
  • Fix immediately, put everything else aside. Time to do some overtime?
